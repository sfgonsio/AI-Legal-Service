# UI TRUTH LABELING RULES

## Purpose
Prevent misleading presentation of system state.

## Required Labels
- Proposed
- Canonical
- Approved
- Rejected
- Conflict
- Missing Support
- Restricted
- AI-Generated Proposal
- Attorney Reviewed

## Rules
- Color alone may not communicate status.
- Proposed content must be visually distinct from canonical content.
- AI-generated support text must be labeled.
- Conflict must not be hidden by summary language.
- Unsupported areas must be visible where relevant.

## Prohibited
- presenting proposal narrative as settled fact
- using certainty language where only support exists
- removing artifact-state labels for aesthetic reasons
