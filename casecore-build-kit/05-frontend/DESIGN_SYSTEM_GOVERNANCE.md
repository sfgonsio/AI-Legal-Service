# DESIGN SYSTEM GOVERNANCE

## Purpose
Define how the design system is governed.

## Rules
- shared status components must be reused, not recreated inconsistently
- proposal/canonical labels must be standardized
- restricted-content styling must be standardized
- design polish may not remove legal-state clarity
- component changes affecting truth/state semantics require coordinated review

## Tokens/Standards
- typography, spacing, color, iconography, elevation, and interaction states should be centralized
- status semantics must map consistently across surfaces

## Rule
The design system is not purely aesthetic; it is part of the truth-communication layer.
