# SAMPLE MATTER FIXTURE SPEC

## Purpose
Define shared sample matters used for validation.

## Fixture Families
### Clean Case
- coherent evidence
- straightforward fact/event mapping

### Conflict Case
- contradictory inputs
- conflict flags expected

### Gap Case
- incomplete support
- missing-support behavior expected

### Restricted Case
- privileged/restricted artifacts
- access/export constraints tested

## Rule
No major workflow claim is considered validated without fixture coverage.
