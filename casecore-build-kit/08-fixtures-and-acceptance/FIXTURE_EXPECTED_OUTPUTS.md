# FIXTURE EXPECTED OUTPUTS

## Purpose
Define expected test behavior for shared fixtures.

## Clean Case Fixture
- at least one document registered
- fact proposals generated
- provenance included
- event candidates generated
- audit entries present

## Future Fixtures
- conflict case
- gap case
- restricted content case

## Rule
No fixture is valid without both input definition and expected-output definition.
