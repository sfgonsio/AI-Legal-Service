# NOTIFICATION POLICY

## Purpose
Define how CASECORE sends or displays notifications.

## Allowed Notifications
- workflow completion
- review requested
- approval required
- export completed
- processing failure requiring attention

## Rules
- sensitive content should not be unnecessarily included in email notifications
- in-app notification is preferred for sensitive workflow state
- notifications must not misstate canonical status
- notification events should be auditable where material
