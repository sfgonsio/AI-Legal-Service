# WORKFLOW SCENARIOS

## Purpose
Document golden path and edge-case behavior.

## Golden Path
1. matter created
2. files uploaded
3. ingestion completed
4. fact proposals generated
5. facts reviewed/promoted
6. event proposals generated
7. event review completed
8. COA mapping produced
9. attorney-facing support generated

## Edge Cases
- OCR failure
- duplicate upload
- missing provenance proposal
- invalid state transition attempt
- restricted artifact export attempt
- taxonomy change after earlier run
- attorney override after prior approval

## Rule
Edge-case behavior must be explicit, not improvised.
