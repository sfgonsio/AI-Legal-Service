# CONTRACT SYSTEM OVERVIEW

## Purpose
Describe how CASECORE contracts are organized.

## Contract Layers
- schemas
- APIs
- events
- enums
- audit rules
- workflows
- prompts
- deterministic program contracts
- bounded agent contracts
- taxonomies
- manifest

## Rule
No implementation component should operate outside the contract system unless explicitly governed and documented.
