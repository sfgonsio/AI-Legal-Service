# Agentic Operating Model SSOT
Status: Active
Purpose: Govern how agents, subagents, and skills are used to build and operate CASECORE under real case pressure.

---

## Core Principle

We are not building a generic platform.

We are building a system that must support a live case moving into deposition.

All design and development must answer:

How does this help win or survive deposition?

---

## Phase Structure

Phase 1: Execution Control
- Migration Manager
- Release Gatekeeper

Phase 2A: Foundational Build Agents
- Legal Architect
- Workflow Governor
- Frontend Builder
- Backend Builder

Phase 2B: Deposition-Driven Product Agents
- Attorney Reviewer
- Deposition Strategy Reviewer
- Evidence Traceability Reviewer

Phase 3: War Room Expansion
- Scenario simulation
- Contradiction engine
- Deposition planning

Phase 4: MCP Integration
- Video/audio ingestion
- External APIs

---

## Agent Model

Agent = role owner  
Subagent = internal specialization  
Skill = repeatable method  

---

## Non-Negotiables

- No file creation outside repo
- No commit without verification
- No legal claims without traceability
- No UI implying certainty without evidence
