# Purpose
Convert UI intent into explicit implementation targets.

# Inputs
- approved design markdown or screenshots
- discrepancy notes
- workflow context

# Process
- identify UI objects affected
- map each object to workflow purpose
- separate frontend, backend, and legal-traceability implications
- produce a concise implementation brief

# Outputs
- object-by-object change brief
- dependencies
- risks
- recommended next agent
